
**********************
Suggested Improvements
**********************

------------------------------------------------------------------------------------------------------------
1. Resilience & Reliability
Automatic queue resume after crash
If the app crashes mid-queue, items stuck in active status are never recovered. Add a startup check that resets any active items back to pending so the queue resumes cleanly on restart.

Retry failed applications
Currently error items sit permanently in the queue with no retry path. Add a "Retry failed" button that resets selected items to pending so users don't have to re-add jobs manually.

Extension health check before queue start
The queue fails mid-run if the extension is disconnected. Add a preflight check on "Start Applying" — validate the extension is connected and LinkedIn is open before the first job is processed, not after.

Graceful LinkedIn session expiry handling
If LinkedIn logs the user out mid-run, the queue currently errors on every subsequent job. Detect the login redirect and pause the queue with a clear "LinkedIn session expired — please log back in" message.

Partial form submission recovery
If a form is partially filled and the app crashes, the next run re-opens the same job and starts from scratch, potentially submitting a duplicate. Track in-progress application state so resumption skips already-submitted applications.

------------------------------------------------------------------------------------------------------------
2. User Experience
Real-time progress within a single application
The UI shows the current job title but not which step it is on ("Uploading resume", "Filling page 2 of 4"). Surface the phase progress from apply-queue-runner.ts's onProgress callback into the UI.

Daily cap progress indicator
Users set a daily cap but have no visible progress bar showing "7 of 10 applied today". Surface todayApplicationCount() in the dashboard.

Queue reordering
Users can add jobs to the queue but cannot reorder them. A drag-and-drop queue with priority support would let users promote high-priority roles to the front.

Answer bank review before queue start
Before running the queue, surface a preview of all cached answers that will be used so users can correct bad AI-generated answers before they are submitted to employers.

One-click "Apply to all search results"
Currently users search, then manually add jobs to the queue. An "Add all to queue" button with a match score threshold filter (e.g., only add jobs scored ≥7) would reduce friction significantly.

Application success rate analytics
The history panel stores outcomes but does not surface aggregated analytics: success rate by company type, by job title keyword, by day of week, pass-through rates on screening questions. A simple dashboard would help users tune their strategy.

------------------------------------------------------------------------------------------------------------
3. LinkedIn Safety & Account Protection
LinkedIn activity rate limiter with adaptive backoff
Currently delays are user-configured. Consider an adaptive mode that automatically reduces speed if LinkedIn starts returning CAPTCHAs or verification screens, and speeds back up when the session is clean.

Weekly application volume warning
LinkedIn's informal threshold for "suspicious" behaviour is roughly 50–100 Easy Applies per week. Add a weekly counter with a configurable warning threshold so users know when they are approaching risky volume.

Human behaviour scoring
Log each session's timing variance, scroll events, and dwell times. Surface a "naturalness score" so users understand how human-like their automation appears.

Cookie/session health monitor
Before each queue run, silently check that the LinkedIn session is still valid and not showing a restricted-account banner. Surface a health status badge in the UI.

------------------------------------------------------------------------------------------------------------
4. Application Quality
Cover letter generation
Many Easy Apply forms include an optional cover letter field. Currently this is either left blank or filled generically. Add a per-job cover letter generated by Claude using the job description and the user's resume highlights.

Pre-submission answer review mode
Add an optional "review before submit" mode where the app fills the form but pauses before clicking Submit so the user can check AI-generated answers on complex questions.

Application scoring feedback loop
Allow users to mark applications as "Got interview" or "Rejected". Feed this signal back into the job scorer so future job matching improves based on what actually worked.

Duplicate application detection
Before adding a job to the queue, check application history for the same company + title + URL. Warn the user if they are about to apply to the same role twice.

------------------------------------------------------------------------------------------------------------
5. Outreach Quality
LinkedIn connection acceptance tracking
After a connection request is sent, there is no follow-up on whether it was accepted. Add an acceptance-rate tracker that checks pending requests weekly and logs acceptance vs. expiry.

Message A/B testing
Allow users to define two message variants per campaign and automatically split-test them. Track response rates per variant to surface which messages perform better.

Sequence pause on reply detection
If a prospect replies to a connection message, the follow-up sequence should automatically pause for that contact. Currently follow-ups fire regardless of whether the person already responded.

CSV validation and preview
Before an outreach campaign starts, validate the CSV for missing required fields, malformed URLs, and duplicate entries. Show a preview table so users catch errors before 500 messages go out.

------------------------------------------------------------------------------------------------------------
6. Data & Persistence
Export to CSV / Google Sheets
Application history and outreach logs are stored locally with no export. A one-click CSV export (or Google Sheets sync) would let users track their job search in their own tools.

Cloud backup of queue and history
Firebase is already integrated for auth. Optionally sync application history and queue state to Firestore so users can restore data after reinstalling the app or switching machines.

Schema migrations for persisted JSON
As noted in the architecture analysis, persisted files have no version field. Adding versioned migrations prevents silent data loss when the app updates and the schema changes.

Settings import/export
Power users who reinstall the app or run it on multiple machines have no way to transfer their settings, answer bank, and screening cache. A single-file export/import would solve this.

------------------------------------------------------------------------------------------------------------
7. Developer & Operational
End-to-end test suite against a mock LinkedIn
The extension and main process have no integration tests. A mock HTML server that mimics LinkedIn's Easy Apply DOM would let the team run the full apply pipeline in CI without a real LinkedIn account.

Extension version compatibility check
When the extension and desktop app are out of sync, automation breaks in hard-to-diagnose ways. Add a strict version compatibility handshake on bridge connect — surface a clear upgrade prompt rather than mysterious failures.

Structured telemetry dashboard
Sentry captures errors but not usage patterns. Add structured events (queue started, application submitted, outreach sent) to a lightweight analytics backend so the team can see which features are actually used.

Changelog / release notes in-app
Users currently have no visibility into what changed between versions. An in-app changelog (shown once on first launch after update) reduces support requests and builds trust.
